# 넥슨 프론트엔드 기술 면접 후기

넥슨은 면접을 여러 번 봤었는데, (가칭)Face play라는 게임 앱의 프론트엔드팀과 jQuery 레거시 코드를 다루고 있는 웹 프론트엔드 팀이었다.
또 다른 한 팀은 FE개발자가 한 명인 팀이었던 것 같다. 
둘다 코딩테스트가 있었고 무난하게 풀 수 있었던 것 같다.
플랫폼은 코딜리티였던 것 같다.
예전에 라쿠텐도 해당 플랫폼에서 코딩테스트를 봤었어서 익숙했다.

## 1. 면접 질문 및 코멘트

- 자기소개
- 일본에 갈 생각은 어떻게 하게 되었는지 - FacePlay팀
- 언어가 힘들지는 않았는지 - FacePlay팀
- 우리팀이 무슨 일을 할 것 같은가 - FacePlay팀
- 왜 React를 사용해야 하는지? 장점 - FacePlay팀
- React를 사용해야 하는데 jQeury를 사용하라고 하면 어떻게 할 것인가? - FacePlay팀
- 랜덤 알고리즘을 구현해봐라 (대답을 잘 못하니 조금씩 같이 질문과 대답 형식으로 진행) - FacePlay팀
- 이력서에 써 놓으신 걸 다 하셨다는 게 믿기지가 않는다 - Legacy 팀
- jQuery를 헤애 하는데 괜찮은지 - Legacy 팀
- CSS에서 중앙 정렬 하는 방법 5가지를 이야기 해달라 - Legacy 팀
- 블로그에 구현해놓은 애니메이션 중 하나를 찝어 구현 방법을 질문 - Legacy 팀
- CSS는 프론트엔드 개발자의 기본이다 - Legacy 팀
- CI는 알겠다, CD는 뭘 의미하는지 - Legacy 팀
- 클로저는 무엇인가 - Legacy 팀
- Dynamic Filter는 어떻게 구현했는가 - Legacy 팀
- 프로젝트의 성능 최적화를 하는 3가지 방법 - Legacy 팀
- DB 구축하는 방법을 설명해라 - 1인FE개발팀
- FE개발자를 추가로 뽑을 생각이 없는데 혼자 어느정도까지 커버할 수 있는가 - 1인FE개발팀
- 웹서비스를 새로 구축해야 할 때 프론트부터 백엔드까지 어떻게 전체적으로 구성해서 올릴 것인가 - 1인FE개발팀

## 2. 면접 후기

- 면접관은 각각 FacePlay팀 한분, Legacy팀 세분(개발자2, 리더1), 1인FE개발팀 세분(인사1,팀장1,개발자1)이었다.
- 전반적으로 프론트엔드 개발 면접을 보면서, 알고리즘, 풀스택 개발, 퍼블리싱 관련 여러 방면의 질문을 받았고 브라우저나 FE 기술 스택에 대한 질문은 잘 없었던 것 같다.
- 연혁이 오래 된 만큼 jQuery를 사용하는 프로젝트와 최신 기술 사이에서 마이그레이션에 대해 많이 고민하고 있는 것 같았다.
